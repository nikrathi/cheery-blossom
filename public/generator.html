Main Command

Run the whole build in one step

This is the quickest way to produce a full ticket set and update the caller data for the site.

Full build

Python entrypoint

python3 scripts/tambola_ticket_service.py build

Generates the manifest, CSV, validation report, website app data, and rendered ticket images.

Wrapper

Shell shortcut

./scripts/build_tickets.sh

Runs the same build flow through the wrapper script.

Useful Commands

Run only the part you need

Generate manifests only

python3 scripts/tambola_ticket_service.py generate

Writes the JSON, Markdown, and CSV ticket manifests.

Validate an existing CSV

python3 scripts/tambola_ticket_service.py validate

Checks ticket uniqueness, number/name consistency, category balance, and frequency rules.

Export caller app data

python3 scripts/tambola_ticket_service.py export-app-data

Rebuilds public/app-data.json from the CSV so the Tambola page uses the new ticket set.

Skip image rendering

python3 scripts/tambola_ticket_service.py build --no-render

Useful if you only need the CSV, validation report, and caller data.

Workflow

How the generator and caller fit together

Step 1

Generate the ticket set

Run the Python service to create a fresh balanced ticket manifest with 30 unique tickets.

Step 2

Validate everything

Make sure every ticket has 16 unique entries, category balance, correct name-to-number mapping, and no duplicate tickets.

Step 3

Export the caller data

The service updates the website data so the Tambola page uses the same ticket set when it searches for random valid call orders.

Step 4

Run the game

Open the Tambola page and start a new game. The caller searches for a fresh random order with one earliest winner per prize.
